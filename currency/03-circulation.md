# 通貨の流通 — 評価圏アクセスと累積の設計

> **v0.8.0** | [← 変換レート](02-conversion-rates.md) | [← README](../README.md)

## 1. 通貨の流通サイクル

```
[ 入力差分の実行 ]
        ↓  通貨：消費（実行コスト）
[ 出力差分の発生 ]
        ↓
[ 評価圏の認識 ]
        ↓  通貨：生成（変換レート × 差分量）
[ 通貨の蓄積 ]
        ↓
[ 評価圏アクセスへの投入 ]  ← 主要用途
        ↓  通貨：消費（アクセスコスト）
[ 新評価圏での実行 ]
        ↓
        └─────────────────── ループ（拡張再生産）
```

差体は通貨を使って新しい評価圏に参入し、より多くの変換機会を獲得する。  
これが通貨の最大化を追求することの意味である。

---

## 2. 評価圏アクセス（主要用途）

新しい評価圏への参入は通貨の消費によって行われる。

### アクセスコストの構造

```
参入コスト = 評価圏のアクセスコスト基準値
             × (1 − 既存の関連所属変換係数の寄与)
             × (1 − 象徴資本補正)
```

| 低減要因 | 内容 |
|---|---|
| 関連所属の設定差分 | 既存所属の設定差分がターゲット評価圏の目標差分水準に近いほどコストが低い |
| 象徴資本保有者との接続 | 評価圏内の象徴資本保有者からの推薦・認定でコストが下がる |
| 中期的時間係数の蓄積 | 差分統合量の累積が深いほど参入障壁が下がりやすい |

### アクセスコストの位置づけ

アクセスコストは「評価圏に参入する価値があるか」の判断基準にもなる。

```
アクセスコスト < 期待される変換収益（変換効率係数 × 差分ギャップ）
  → 参入するインセンティブあり

アクセスコスト > 期待される変換収益
  → 別の評価圏への展開・または既存評価圏の深化を優先する
```

## 3. 展開戦略パターン

差体が取りうる通貨運用の戦略。プラズミズム状態と連動する。

| パターン | 内容 | 通貨の動き | 適した適応状態 |
|---|---|---|---|
| **深化型** | 既存評価圏での変換効率を高め、通貨密度を上げる | 収入増・支出安定 | 液体〜気体 |
| **展開型** | 通貨を使って新評価圏にアクセスし、変換経路を広げる | 支出後に収入増 | 気体〜プラズマ |
| **再建型** | アクセスコストの低い評価圏から通貨を再蓄積する | 低支出・収入回復 | 固体〜液体 |
| **並行型** | 複数評価圏を同時に運用し、複利的な差分統合量を蓄積する | 高収支・高変動 | プラズマ |

固体状態では再建型から始め、差分統合量を蓄積しながら深化型・展開型へ遷移していくことが基本経路となる。

## 4. 差分統合量との接続

通貨の生成・消費サイクルの累積が、プラズミズムの状態遷移トリガーである差分統合量の実態的な記録となる。

```
差分統合量 ≈ Σ（通貨生成イベント × 評価圏接触深度 × 変換効率係数）
```

差分統合量は通貨量そのものではなく、通貨の循環サイクルの**深度と質の累積**である。

```
ケースA: 低効率な評価圏で通貨を10回生成（各1単位）
  差分統合量への貢献 = 10 × 低変換効率係数 = 小

ケースB: 高効率な評価圏で通貨を1回生成（1単位）
  差分統合量への貢献 = 1 × 高変換効率係数 = 大
```

通貨の最大化と差分統合量の最大化は同一方向を向いているが、完全に一致はしない。  
後者は「接触の深さ」を重視するため、量より質のサイクルが差分統合量の向上に寄与する。

> → 差分統合量の定義: [locus/01-plasmism.md § 3](../locus/01-plasmism.md)

## 5. 通貨の減価

通貨に時間的な減価（decay）を設けるかは差体の設計による。

| 設計 | 意味 | 特性 |
|---|---|---|
| **減価あり** | 未使用通貨は時間とともに価値が下がる | 行動の継続を促す。停滞を可視化できる |
| **減価なし** | 通貨は蓄積されるだけ（デフォルト） | シンプル。長期的蓄積が明示される |

### 所属中央値の動的変化との接続

Straでは通貨の減価を「設定差分の陳腐化」として解釈できる。

```
時間の経過
  → 所属の中央値が上昇（社会の平均水準が向上）
  → 以前の設定差分値が相対的に縮小（差分ギャップが拡大）
  → 既存通貨の「購買力」（変換効率）が低下
  → 継続的な実行と更新がないと通貨の実効価値は減衰する
```

これは通貨の明示的な減価設計がなくても、中央値の動的変化が実質的な減価として機能することを意味する。

## 6. 複式モデルの日次実践との接続

通貨の複式モデルは `locus/05-task-execution.md` の差分実行サイクルと以下のように対応する。

| タスク実行の要素 | 通貨の複式モデルへの対応 |
|---|---|
| タスク（入力差分の最小単位） | 通貨の消費単位 |
| 見積もり時間 | 期待消費通貨量 |
| 実績時間 | 実際の消費通貨量 |
| 出力差分の観測 | 通貨回収量の確認 |
| 夜の統合フェーズ | 日次のポジション決済・未回収ポジションの整理 |
| 週次統合レビュー | 週間収支の確認・変換定義の補正 |
| 月次戦略レビュー | 変換定義マップの大幅更新 |

通貨の観点からTaskShootを運用すると、「今日のタスクリスト＝今日の通貨投資計画」となり、  
実行の選択に対する経済的な根拠が明確になる。

## 7. 通貨ポジションの健全性指標

差体が自身の通貨状態を評価するための観測指標。

| 指標 | 意味 | 健全な状態 |
|---|---|---|
| **日次収支** | 1日の通貨生成量 − 消費量 | プラス（または意図的マイナス） |
| **未回収ポジション率** | 全投資中のポジションの割合 | 30〜60%（過多でも過少でもない） |
| **評価圏別変換効率** | 各評価圏での回収率の推定 | 複数圏でバランス良く生成できる |
| **アクセスコスト残高比** | 次の目標評価圏のアクセスコスト / 通貨残高 | 0.5以下（参入できる余力がある） |

## 注記

通貨の最大化を追求することは推奨されるが、通貨 ≠ 豊かさである。  
豊かさの5条件のうち「③ 選択肢の存在」と「⑤ 実行基盤」は通貨の蓄積と連動しやすいが、  
「④ 意志の保持」は通貨量では担保されない。通貨のポジション管理が主体の意志を代替することはない。

> → 豊かさの5条件: [locus/00-principles.md § 3](../locus/00-principles.md)  
> → 通貨の定義: [currency/01-currency-definition.md](01-currency-definition.md)
