## 1. 変換レートの構造

一つの実行から生成される通貨量は、以下の4変数の積として記述される。

```
Δ通貨（一実行あたり）
  = 実行量
    × 所属変換係数（所属ごとに定義）
    × 評価圏変換係数（評価圏ごとに定義）
    × プラズミズム係数（差体の現在の適応状態）
    × 時間係数（時間層・実行タイミング）
```

各係数は差体が**主観的に推定する**。絶対的な正確さは不要であり、実行後のフィードバックによるベイズ的補正を前提とする。

> → 係数の補正プロセス: [§6. 推定と補正](#6-推定と補正)

## 2. 所属変換定義

各所属が保持する変換パラメータ。差体は参加している所属ごとにこれを推定・記録する。

| パラメータ | 定義 | 設定方法 |
|---|---|---|
| **対応資本タイプ** | この所属での実行が何資本に変換されやすいか | 評価圏の観測から推定 |
| **設定差分値** | 現在の所属内での差分量（現在地） | 自己測定・外部フィードバック |
| **基礎変換レート** | 設定差分1単位あたりの通貨生成基準値 | 過去の出力差分観測から推定 |
| **飽和閾値** | 同一所属への過剰投入で変換効率が低下し始める点 | 経験的に推定 |
| **差分ギャップ** | 目標差分水準 − 現在の設定差分 | [locus/03-causal-graph.md](../locus/03-causal-graph.md) より |

### 飽和閾値について

```
変換効率
   ↑
1.0 ┤          ╭──────── 最大効率帯（差分ギャップ > 0）
    │         ╱
    │        ╱
    │       ╱
0.0 ┤──────╯─────────────── 飽和（差分ギャップ ≦ 0）
    └──────────────────────────────────── 投入量 →
          飽和閾値
```

差体の設定差分が目標差分水準を超えた所属では変換効率が低下する。  
より差分ギャップの大きい所属・評価圏へ投入をシフトするシグナルとして機能する。

### 所属間の両替

複数所属の変換定義を持つ差体は、所属ごとの基礎変換レートの比を**両替レート**として換算できる。

```
両替レート（所属A → 所属B）= 所属Bの基礎変換レート ÷ 所属Aの基礎変換レート
```

両替レートは固定ではなく、各所属の設定差分・差分ギャップ・飽和状態によって変動する。

## 3. 評価圏変換定義

各評価圏が保持する変換パラメータ。差体は参入している評価圏ごとにこれを観測・記録する。

| パラメータ | 定義 | 備考 |
|---|---|---|
| **目標差分水準** | 変換が成立する最低閾値 | [sphere/03-evaluation-capital.md § 10](../sphere/03-evaluation-capital.md) |
| **変換効率係数** | 閾値超過後の単位あたり通貨変換率 | 評価圏の密度・競争度に依存 |
| **象徴資本補正** | 象徴資本保有者によるレート操作幅 | 非対称な変換効率の記述 |
| **アクセスコスト** | 新規参入に必要な通貨量 | 高密度・高競争の評価圏ほど高い |

### 象徴資本と変換レートの非対称性

象徴資本保有者は目標差分水準の設定権を持つ（`sphere/04-symbolic-capital.md`）。  
これは通貨の変換レートにおける**レート設定権**として作用する。

```
象徴資本保有者のいる評価圏での変換レート:
  保有者の設定差分が認定される → 高い変換効率係数が適用される
  非保有者の設定差分は閾値判定が厳しくなる → 実質的な変換効率が低下する
```

象徴資本の蓄積は、将来的にアクセスコストの低減・変換効率係数の優遇という形で通貨に反映される。

> → 象徴資本の定義: [sphere/04-symbolic-capital.md](../sphere/04-symbolic-capital.md)

## 4. プラズミズム係数

差体の現在の適応状態が変換全体に乗算する効率係数。

| 適応状態 | 係数の目安 | 意味 |
|---|---|---|
| **固体** | 0.3〜0.5 | 外部フィードバックの経路が狭く、評価圏認識が困難。通貨回収率が低い |
| **液体** | 0.6〜0.8 | 変換経路は開いているが設計精度・観測精度が低い |
| **気体** | 1.0〜1.3 | 効率的な変換。複数評価圏の並行運用が可能 |
| **プラズマ** | 1.5〜2.0 | 変換経路の自律的な開拓・評価圏の変容まで可能 |

同じ実行でも適応状態によって回収通貨量が3〜6倍変動しうる。  
プラズミズム状態の向上が通貨効率の最大のレバレッジとなる。

> → プラズミズムの定義・状態遷移: [locus/01-plasmism.md](../locus/01-plasmism.md)

## 5. 時間係数

時間の3層モデル（`sphere/02-belonging-median.md § 5`）と連動し、実行タイミングが通貨変換に影響する。

| 時間層 | 係数の作用 |
|---|---|
| **短期（日次〜週次）** | 機会タイミング係数。評価圏の認識窓が開いている時間帯は係数が高い。逆に窓が閉じていると出力差分が不可視になる |
| **中期（月〜年）** | 累積複利係数。差分統合量の蓄積が深いほど同一実行に対する変換効率が上昇する |
| **長期（差分の歴史化）** | 象徴資本形成係数。持続的な認識の蓄積がアクセスコストを低減し、将来の変換効率を構造的に向上させる |

### 短期タイミングの例

```
同一の実行（例：論文の公開）

  評価圏の認識窓が開いている時期（学会・採用季節等）
    → 時間係数: 1.2〜1.5
    → 出力差分が認識されやすい

  評価圏の認識窓が閉じている時期
    → 時間係数: 0.5〜0.8
    → 同じ出力差分が不可視のままになりやすい
```

## 6. 推定と補正

変換レートの正確な事前計算よりも、実行後のフィードバックによる継続的な補正が重要である。

```
初期推定（仮置き）
    ↓
実行 → 出力差分の観測
    ↓
回収通貨 vs 期待通貨の差分
    ↓
係数の補正（ベイズ的更新）
  ├─ 基礎変換レートの修正（所属変換定義の更新）
  ├─ 変換効率係数の修正（評価圏変換定義の更新）
  └─ 時間係数のパターン学習
    ↓
次の実行へ
```

このサイクルは `locus/04-deviation-design.md` の差分設計プロセス（Step 7: 実行・観測・補正ループ）と構造的に同一である。
